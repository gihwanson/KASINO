# AI 댓글 품질 개선 가이드

## 🎯 가장 효율적인 학습 방법

### 1. **Few-shot Learning (가장 효과적)**
   - 실제 좋은 댓글 예시를 추가하면 AI가 그 스타일을 학습합니다
   - 예시가 많을수록 더 정확하게 학습합니다
   - **효과**: 즉시 반영, 높은 정확도

### 2. **나쁜 예시 추가**
   - 형식적이거나 부적절한 댓글을 피하도록 학습
   - AI가 피해야 할 패턴을 명확히 인식

### 3. **프롬프트 개선**
   - 피드백을 받아 프롬프트를 직접 수정
   - AI 조교가 개선 방법을 제안

---

## 📋 사용 방법

### 1. 학습 프로그램 실행
```
AI_학습_시스템.bat 실행
```

### 2. 명령어

#### `테스트`
- 게시글 본문과 기존 댓글을 입력하여 AI 댓글 생성 테스트

#### `좋은예시`
- 생성된 댓글이 좋으면 "좋은예시" 명령어로 저장
- Few-shot learning에 자동으로 추가됨

#### `나쁜예시`
- 나쁜 댓글을 입력하고 이유를 설명
- AI가 피하도록 학습

#### `프롬프트`
- 프롬프트를 직접 수정하거나 AI 조교의 제안을 받음

#### `히스토리`
- 학습 히스토리 확인

## 🔄 자동 반영 시스템

### 학습 데이터 저장
- `ai_learning_data.json` 파일에 자동 저장
- 다음 항목들이 저장됩니다:
  - 좋은 댓글 예시 (Few-shot learning)
  - 나쁜 댓글 예시 (피해야 할 패턴)
  - 개선된 프롬프트
  - 피드백 히스토리

### 매크로봇 자동 로드
- `실행하기.bat` 실행 시 자동으로 학습 데이터를 불러옴
- Few-shot 예시가 프롬프트에 자동 포함됨
- 개선된 프롬프트가 자동으로 사용됨

## 💡 학습 예시

### 좋은 예시 추가
```
1. 테스트 명령어로 댓글 생성
2. 생성된 댓글이 좋으면 "좋은예시" 입력
3. 좋은 댓글 입력 (또는 생성된 댓글 사용)
4. 자동으로 Few-shot learning에 추가됨
```

### 나쁜 예시 추가
```
1. "나쁜예시" 명령어 입력
2. 나쁜 댓글 입력 (예: "좋은 글 감사합니다")
3. 이유 입력 (예: "형식적이고 감사 단어 사용")
4. AI가 피하도록 학습됨
```

## ⚙️ 기술적 세부사항

### Few-shot Learning
- 최대 5개의 좋은 예시를 프롬프트에 포함
- 각 예시는 본문, 기존 댓글, 좋은 댓글로 구성
- AI가 이 패턴을 학습하여 유사한 댓글 생성

### 프롬프트 구조
```
기본 프롬프트
+ Few-shot 예시 (좋은 댓글)
+ 나쁜 예시 (피해야 할 패턴)
+ 개선된 프롬프트 (있는 경우)
```

### 자동 반영
- `macro_bot.py`의 `__init__`에서 학습 데이터 자동 로드
- `generate_ai_comment` 함수에서 Few-shot 예시 자동 포함
- 사용자 개입 없이 자동으로 적용

## 📝 주의사항

1. **예시 품질**: 좋은 예시를 추가할수록 품질이 향상됩니다
2. **예시 개수**: 너무 많으면 프롬프트가 길어질 수 있으니 최대 5개 권장
3. **정기적 업데이트**: 새로운 좋은 댓글을 발견하면 계속 추가하세요

## 🎓 학습 팁

1. **다양한 상황의 예시 추가**
   - 존댓말 본문 → 존댓말 댓글
   - 반말 본문 → 반말 댓글
   - 기쁜 글, 슬픈 글 등 다양한 감정

2. **나쁜 예시도 중요**
   - "감사합니다" 같은 형식적 댓글
   - 본문과 무관한 댓글
   - 말투가 안 맞는 댓글

3. **피드백 활용**
   - AI 조교에게 질문하면 개선 방법 제안
   - 프롬프트를 직접 수정하여 미세 조정
